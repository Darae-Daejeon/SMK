## 6. 제작 방식

- 메일 본문 기준 폭은 `600px`로 설정한다.
- 레이아웃은 이메일 호환성이 높은 프레젠테이션용 `<table>` 구조로 작성한다.
- 주요 시각 영역은 PNG 이미지로 고정한다.
- 각 기술 카드는 별도 이미지로 분리하고 `<a>` 요소로 감싸 개별 링크를 연결한다.
- 연락처와 보조 설명은 HTML 텍스트로 작성해 이미지 차단 환경에서도 확인 가능하게 한다.
- 핵심 스타일은 인라인으로 작성한다.
- 자바스크립트, 외부 폰트, CSS 그리드, 플렉스 레이아웃은 사용하지 않는다.
- 이미지맵은 메일 클라이언트별 호환성이 낮으므로 사용하지 않는다.

## 7. 이미지 원칙

- 기존 ETRI SMK에 포함된 원본 기반 대표 이미지를 우선 재사용한다.
- 기술별 이미지의 높이는 콘텐츠 분량과 가독성에 맞춰 조정한다.
- HTML의 `width`, `height` 속성에 실제 이미지 비율을 반영한다.
- 모든 이미지에 기술 내용을 설명하는 대체 텍스트를 입력한다.
- 이미지 파일은 GitHub Pages에서 불러올 수 있는 상대경로로 연결한다.
