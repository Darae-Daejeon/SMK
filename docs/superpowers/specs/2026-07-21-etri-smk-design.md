# ETRI 기술 SMK 제작 설계

## 1. 목표

- 다운로드 폴더의 ETRI 기술자료 4건을 근거로 기술별 SMK HTML을 제작한다.
- 기존 `catholic/레퍼런스-smk.html`의 시각 체계와 정보 구조를 재사용한다.
- 각 HTML을 A4 가로 방향으로 인쇄했을 때 한 장에 정확히 출력되도록 한다.

## 2. 산출물

- `ETRI/tech-01.html`
- `ETRI/tech-02.html`
- `ETRI/tech-03.html`
- `ETRI/tech-04.html`
- `ETRI/index.html`: 기술 4건의 선택·열람 화면
- `ETRI/assets/`: 기술자료에서 추출한 도면 및 이미지

## 4. 콘텐츠 구성

- 기술명과 한 줄 핵심 가치
- 기술 개요
- 기존 방식의 한계와 기술 차별성
- 핵심 기능 또는 기술 구성
- 적용 분야와 예상 수요처
- 도입 효과
- 원본 자료에서 선별한 기술 도면, 절차도 또는 성능 결과
- ETRI 기술이전 페이지와 기술번호

## 5. 표현 원칙

- 원본 자료의 기술 범위와 표현을 우선한다.
- 전문용어는 비전문가가 이해할 수 있도록 짧게 풀어 쓴다.
- 근거가 없는 시장규모, 성능 우위, 사업화 단계는 추가하지 않는다.
- 시장수치와 TRL은 원문에서 확인된 경우에만 출처와 한계를 함께 표기한다.
- 기술별 문안은 개조식으로 작성한다.

## 6. 디자인 원칙

- 기존 가톨릭대 SMK의 보라색 계열, 제목 위계, 정보 블록 구조를 유지한다.
- 화면 비율은 16:9로 제한하지 않는다.
- 기준 캔버스는 A4 가로 `297mm × 210mm`로 설정한다.
- 화면에서는 종이 전체가 중앙에 맞춰 보이도록 반응형 축소한다.
- 기술별 도면은 원본 자료에서 추출하고, 용도에 맞게 자르되 의미를 훼손하지 않는다.

## 7. 인쇄 규격

- CSS `@page`를 `size: A4 landscape; margin: 0;`으로 설정한다.
- 실제 문서 요소를 `width: 297mm; height: 210mm;`로 고정한다.
- 문서 내부에 상하좌우 동일한 `8mm` 안전여백을 둔다.
- 인쇄 시 그림자, 화면 배경, 확대·축소용 변환을 제거한다.
- `print-color-adjust: exact`로 배경색과 그래픽 색상을 유지한다.
- 페이지 나눔을 차단해 각 기술 SMK가 한 페이지만 출력되도록 한다.
- 브라우저 인쇄 배율 100%, 머리글·바닥글 해제 조건에서 검증한다.

## 8. 검증 기준

- 기술별 HTML을 Chromium에서 A4 가로 PDF로 출력한다.
- PDF가 정확히 1페이지인지 확인한다.
- 페이지 크기가 A4 가로인지 확인한다.
- 상하좌우 내부 여백이 동일한지 확인한다.
- 텍스트 잘림, 겹침, 불필요한 줄바꿈이 없는지 확인한다.
- 도면의 글자와 수치가 인쇄물에서 식별 가능한지 확인한다.
- `file://`로 열었을 때도 주요 내용과 이미지가 정상 표시되는지 확인한다.

## 9. 범위 제외

- 원본 PPTX와 PDF의 수정
- 근거 없는 외부 시장조사 수치 추가
- 별도 PPTX 또는 압축파일 제작
- 웹 서버나 데이터베이스 도입
