## 1. 디자인 요약

- 성격: 독립 크리에이티브·디지털 프로덕션 스튜디오 포트폴리오
- 핵심 인상: 다크, 절제, 편집 디자인, 타이포그래피 중심, 실험적 인터랙션
- 주요 대비: 검정에 가까운 배경과 밝은 회색 타이포그래피
- 핵심 장치: 초대형 `DEADWATER` 워드마크, 좁고 긴 Condensed 서체, 프로젝트 미리보기, 스크롤 기반 전환
- 정보 구조: 소개 → 프로젝트 → 전문 분야 → 팀 → 문의
- 장식 최소화: 카드 테두리·그림자·그라디언트보다 크기, 여백, 색 대비, 움직임으로 위계 형성

## 3. 디자인 토큰

### 3.1 색상

| 토큰 | 값 | 용도 |
|---|---:|---|
| `--color-gray-dark` | `#111111` | 기본 페이지 배경 |
| `--color-black` | `#000000` | 프로젝트 미디어, 오버레이, 깊은 배경 |
| `--color-gray` | `#CFCFCF` | 기본 본문, 대형 워드마크, 기본 링크 |
| `--color-gray-medium` | `#3F3F3F` | 비활성 프로젝트명, 보조 설명, 직책 |
| `--color-white` | `#FFFFFF` | 강조 문장, 활성 상태, 주요 이름 |

- 기본 배경과 본문색의 조합을 전 페이지에 일관되게 적용
- 프로젝트 이미지 내부의 브랜드 색상은 원본 이미지에만 허용
- UI 크롬은 무채색 유지

### 3.2 서체

| 역할 | 서체 | 크기·굵기 기준 |
|---|---|---|
| 본문·내비게이션 | `Inter`, `Arial`, sans-serif | 14px/21px, 400 |
| 서비스 설명 | `Inter` | 16px/25.6px, 400 |
| 카드·인물 제목 | `Inter` | 24px/28.8~31.2px, 300~400 |
| 소개 강조문 | `Inter` | 데스크톱 40px/52px, 200, `-0.8px` |
| 프로젝트 제목 | `Humane`, 좁은 Condensed 대체서체 | 80px/72px, 500, 대문자 |
| 대형 CTA | `Humane` | 데스크톱 184px/128.8px, 400, 대문자 |
| 메인 워드마크 | `Humane` 계열 | 뷰포트 폭에 반응하는 초대형 대문자 |

- `Humane`은 브랜드 인상을 결정하는 핵심 서체
- 라이선스 또는 원본 폰트 확보가 어려우면 `Roboto Condensed`, `Archivo Narrow`, `Arial Narrow`보다 폭이 더 좁은 디스플레이 서체 우선 검토
- 대체서체 사용 시 글자 폭과 전체 블록 폭을 기준으로 재조정

## 5. 인터랙션과 모션

### 5.1 프로젝트 미리보기

- 데스크톱에서 프로젝트명 호버 또는 포인터 접근 시 미리보기 활성화
- 활성 프로젝트명 색상을 `#3F3F3F → #FFFFFF`로 전환
- 미리보기는 우측 상단 또는 포인터 인접 영역에 고정
- 이미지·영상 프레임 비율은 원본 프로젝트별 유지
- 미리보기 내부에 닫기, 재생, 프로젝트 보기 컨트롤 제공 가능

### 5.2 스크롤

- 섹션 진입 시 텍스트와 미디어가 지연 등장
- 히어로와 미디어가 스크롤 중 일시 고정되는 인상 사용
- 원본에서 확인된 easing: `cubic-bezier(.165, .84, .44, 1)`
- 기본 전환: 150~300ms
- 대형 미디어 전환: 약 600ms
- 모션 감소 설정에서는 고정·패럴랙스·자동재생 효과 제거

### 5.3 커서와 컨트롤

- 데스크톱 프로젝트 영역에서 기본 커서를 숨기고 커스텀 커서 사용 가능
- 커스텀 커서는 상태에 따라 `Lecture`, `Fermer`, `Voir le projet` 표시
- 모바일은 커스텀 커서 제거
- 모바일 컨트롤은 최소 44 × 44px 터치 영역 확보

## 8. 콘텐츠 규칙

- 제목은 짧고 명사형 중심
- 프로젝트명은 대문자 표시
- 설명은 과장 없이 수행 내용과 결과 중심
- 섹션 레이블은 1~2단어 유지
- 역할과 연도는 프로젝트명보다 한 단계 낮은 위계로 배치
- 프랑스어 원문을 다른 언어로 교체하더라도 줄 수와 블록 폭을 먼저 맞출 것

## 9. 접근성 주의사항

- `#3F3F3F` 텍스트는 `#111111` 배경에서 대비가 낮음
- 비활성 프로젝트명과 직책은 장식 정보에만 사용하거나 대비 상향 필요
- 커스텀 커서만으로 상태를 전달하지 말 것
- 모든 프로젝트 행에 키보드 포커스 상태 제공
- 영상은 음소거 기본, 정지·재생 수단 제공
- `prefers-reduced-motion` 대응 필수
- 초대형 글자가 확대·축소 시 화면을 가리지 않도록 `clamp()` 사용
- 이미지 `alt`는 인물명 또는 프로젝트 맥락을 설명
- 실제 WCAG 적합성은 DOM 의미 구조, 키보드 이동, 스크린리더 테스트로 별도 검증
